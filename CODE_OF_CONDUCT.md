# Code Of Conduct

Aurel welcomes contributors with different experience levels, backgrounds, and goals. The project is especially focused on helping beginners contribute with confidence, so discussions should stay clear, respectful, and practical.

## Short Version

- Assume people are here to learn or improve the project.
- Keep feedback specific and actionable.
- Ask for changes with enough context that a beginner can respond.
- Move sensitive or safety-related concerns to private maintainer channels.

## Expected Behavior

- Be patient with beginner questions and incomplete first attempts.
- Keep feedback specific to the code, documentation, issue, or proposal.
- Explain tradeoffs when asking for changes.
- Credit ideas and work from other contributors.
- Respect maintainers' time by keeping issues and pull requests focused.

## Unacceptable Behavior

- Harassment, threats, personal attacks, or discriminatory language.
- Publicly sharing private contact details or sensitive information.
- Repeatedly derailing issues, reviews, or discussions after maintainers ask you to stop.
- Pressuring maintainers or contributors for unpaid work outside the project scope.

## Reporting Concerns

If you see behavior that makes participation unsafe or unwelcoming, contact the maintainers privately where possible. If the repository has private maintainer contact options enabled on GitHub, use those instead of a public issue.

Include what happened, where it happened, and any context that would help maintainers review the concern. Maintainers should handle reports with care, limit unnecessary disclosure, and take action that fits the situation.

## Maintainer Responsibilities

Maintainers may edit, hide, lock, or remove comments, issues, pull requests, or contributors when needed to protect the project community. Enforcement should be proportionate, documented when appropriate, and focused on keeping Aurel useful and welcoming for contributors.
